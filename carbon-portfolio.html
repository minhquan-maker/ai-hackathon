IDEA #1 — RANKED #1
CarbonMind — AI Carbon Coach
An LLM-powered conversational agent that understands natural-language carbon queries and generates personalized reduction plans grounded in real emission data.
TOP PICK
AI Behavior Change
Problem
Manual carbon calculators require users to know their consumption data (km driven, kWh used, kg of beef eaten). Most people don't have this data memorized. 90% of users abandon carbon apps after the first use because the data entry friction is too high. Existing AI assistants (ChatGPT, Claude) hallucinate emission factors — they don't know that 1 kg of beef ≈ 27 kg CO₂e.
Proposed Solution
A conversational AI that accepts natural language ("I drove 40km in my diesel car and had a beef burger for lunch") and returns a structured emission estimate + a specific reduction plan. Uses RAG (Retrieval-Augmented Generation) with Climatiq's emission factor database to ensure factual accuracy. Learns user context over time via session memory.
Core Technology
AI/ML: Claude API (LLM) with RAG pipeline over Climatiq emission factors + CO2 Signal grid intensity API for location-aware calculations. Data pipeline: user text → intent parsing → emission factor lookup → calculation → natural language response.
Claude API RAG / Vector DB Climatiq API CO2 Signal API React + Vite FastAPI
Differentiation
Unlike generic LLMs (hallucinate facts) or static calculators (require manual entry), CarbonMind combines natural language understanding with grounded emission data. Compared to Klima/Wren (pure calculators), it acts as an ongoing coach, not a one-time tool. Compared to Climatiq Autopilot (B2B), it targets consumers with a friendly interface.
Feasibility
Hackathon MVP is highly achievable. Frontend chat UI (React) + Claude API calls + a curated local emission factor JSON (100-200 common items) can demonstrate the full flow in 24 hours. No complex ML training needed — prompt engineering + RAG is sufficient. Production upgrade: swap local JSON for Climatiq API.

  • LLM API costs scale with usage — need prompt caching and rate limiting for hackathon demo

User opens app → "I took a 25km taxi to the airport, had breakfast at the hotel (eggs + coffee), and my apartment uses gas heating" → CarbonMind responds with: 4.2 kg CO₂e total (taxi: 3.1kg, breakfast: 0.8kg, heating: 0.3kg) + "Your biggest lever is transport. Taking the airport bus instead would save 2.6kg CO₂e per trip."
IDEA #2 — RANKED #2
FoodPrint — AI Meal Carbon Tracker
Snap a photo of your meal and get an instant CO₂e breakdown with AI-identified ingredients and personalized low-carbon alternatives.
RUNNER UP
Food AI Vision Automation
Problem
Food accounts for 20-30% of personal emissions globally, yet most people have no idea which meals are high-carbon. "Local and seasonal" advice is too vague. Beef vs. chicken vs. tofu — the difference is 10x in CO₂ per gram of protein. Manual food logging (MyFitnessPal style) is tedious and requires nutritional database lookup.
Proposed Solution
A mobile/web app where users snap a photo of their meal. A vision model (Claude with vision) identifies ingredients, maps them to a carbon database (Klimato-style food emission factors), and returns an instant CO₂e score with a breakdown. Includes a "Carbon Swap" feature: for each high-carbon item, suggests a lower-carbon alternative from the same cuisine.
Core Technology
AI/ML: Vision model (Claude with vision or GPT-4o vision) for ingredient identification + rule-based emission mapping from a curated food carbon database. No ML training required for MVP.
Claude Vision Food Carbon DB React Native Expo Camera localStorage
Differentiation
Existing food apps track calories/nutrients, not carbon. Klimato is B2B-only (restaurant menus). No consumer app combines visual AI + carbon data + low-carbon alternatives. This bridges the gap between "I care about food carbon" and "I know what to order."
Feasibility
Highly feasible for hackathon. Vision API + curated food carbon JSON (~500 items) = full working demo. Web version: camera access + React. Native mobile: Expo + camera library. Key MVP constraint: limit to ~50 common dishes for accurate AI identification.

Example User Flow
User snaps a photo of a beef lasagna at a restaurant. AI identifies: beef, pasta, cheese, tomato sauce. System calculates: ~5.2 kg CO₂e (vs. chicken lasagna: ~2.1 kg, mushroom lasagna: ~0.8 kg). Shows: "This meal = charging your phone 520 times. The mushroom version has 84% less carbon."
IDEA #3 — RANKED #3
CarbonHive — Gamified Community Challenges
Transform individual carbon reduction into a social, game-like experience with daily micro-challenges, team competitions, and real-world impact visualization.
THIRD PLACE
Community Behavior Change Gamification
Problem
Every carbon app has a 90%+ dropout rate after first use. Knowing your footprint is not enough to change behavior. Social accountability, immediate rewards, and daily micro-actions are proven to sustain behavior change — but no consumer carbon app applies gamification effectively. The Carbon Footprint Camp boardgame proved that game mechanics make carbon education engaging; this extends that to an app.
Proposed Solution
An app/website where users join 7-day micro-challenges (e.g., "No beef this week", "Public transit only", "Meatless Monday x4"). Completing challenges earns Hive Points, unlocks badges, and contributes to a community carbon pool. Teams compete on leaderboards. Progress is visualized as a growing forest (more reductions = more trees planted via partnership with tree-planting orgs).
Core Technology
Frontend gamification engine (React + Framer Motion), challenge state machine, progress tracking. Backend: lightweight (Firebase Auth + Firestore). AI component: AI challenge recommender — analyzes user's past behavior and suggests the highest-impact challenges for their lifestyle (LLM-powered).
React + Vite Firebase Framer Motion Claude API Chart.js
Differentiation
Klima and Wren have social features ("Multiply") but they're shallow. No carbon app applies behavioral economics (commitment contracts, loss aversion, streak mechanics) effectively. CarbonHive builds on the proven Carbon Trader boardgame mechanics and applies them to daily digital life. The AI recommender ensures challenges are personally relevant, not generic.
Feasibility
Best hackathon demo potential. Visual, interactive, immediately understandable. No complex ML needed for MVP — challenge system and leaderboard can be fully frontend. AI recommender is an add-on layer. The forest visualization and badge system make great demo screenshots.

User joins "No Beef Week" challenge (7 days, 420 participants). Daily check-ins via the app. If user slips on Day 3 and eats a hamburger, they can "make it up" by doing a 2km walk instead of driving. By end of week: saved estimated 8.4 kg CO₂e, earned 340 Hive Points, unlocked "Green Warrior" badge, contributed to 0.2 trees planted in Sumatra via partner org.
IDEA #4
TransitIQ — AI-Powered Commute Analyzer
Automatically detect transport mode from smartphone sensors and GPS, calculate real-world emissions, and suggest lower-carbon commute alternatives with live eco-routing.
Transport Passive Tracking AI
Problem
Transport is the largest component of personal carbon footprints in developed countries (30-40%). But calculating it requires knowing vehicle type, fuel efficiency, distance, and occupancy — data most users don't have. Manual logging is impractical for daily commutes.
Proposed Solution
App passively tracks GPS + accelerometer data during commute. ML model classifies transport mode (walk/bike/bus/car/train) from sensor patterns. Distance + mode → emission calculation via emission factor database. Live integration with Google Maps Eco-Friendly Routing to suggest lower-carbon alternatives in real time.
Core Technology
Sensor fusion: GPS + accelerometer for mode detection via a pre-trained Random Forest / TensorFlow Lite model. Google Maps Routes API for eco-routing. Emission calculation from distance + mode.
TensorFlow Lite Google Maps Routes API React Native CoreLocation / Geolib Firebase
Differentiation
Google Maps already has eco-routing, but it doesn't track your ongoing footprint — it's a one-time navigation tool. No consumer app combines passive mode detection + cumulative footprint tracking + live alternative routing. This is the most automation-forward idea in the portfolio.
Feasibility
Medium complexity for hackathon. Mode classification model needs pre-training on a labeled dataset (can use a pre-trained model from research). Google Maps API has a generous free tier (40k req/month). MVP can demo with pre-recorded GPS traces.

IDEA #5
FootprintSync — Bank Transaction Auto-Tracker
Connect your bank account via open banking to automatically categorize purchases and calculate your real, data-driven carbon footprint with zero manual entry.
Automation Data Analytics AI
Problem
The biggest dropout driver in carbon apps is manual data entry. Every existing solution (Klima, Wren, Capture) asks users to log their purchases. Users don't have time, don't know their consumption data, and quit. Open banking in Europe/Singapore/Vietnam (via Stripe Connect, Plaid) now enables automatic transaction categorization — the missing link.
Proposed Solution
User connects their bank via open banking. Transactions are auto-fetched and categorized by an AI model (GPT-4o or fine-tuned classifier) into carbon-relevant categories: transport (fuel, Uber), food (restaurants, groceries), energy (utilities), goods (shopping). Each category maps to emission factors. Weekly/monthly footprint is auto-calculated and updated — no user action required.
Core Technology
Open banking via Stripe Connect (supports Vietnam and 25+ countries) or Plaid. Transaction categorization via LLM or fine-tuned classifier. Emission calculation from categorized transaction data + national emission factors.
Stripe Connect Claude API FastAPI PostgreSQL React Dashboard
Differentiation
Most innovative in terms of zero-friction automation. No other consumer carbon app does automatic transaction-based tracking at scale. Carbonara attempted this but was acquired and shut down. The gap remains open. This would be the first production-ready implementation.
Feasibility
Hardest to demo at hackathon — requires bank connection setup, which needs real bank accounts and user OAuth flows. MVP hackathon demo can simulate with anonymized transaction data. Production requires Stripe/Plaid approval and regulatory compliance.

  • Regulatory compliance (PSD2 in EU, Vietnam's open banking regulations) adds complexity
IDEA #6
SupplyChain Lens — Product Carbon Scanner
Scan a product barcode or QR code to reveal its supply chain carbon footprint, from raw material extraction to factory to your home, powered by publicly available ESG data.
Supply Chain Transparency AI
Problem
Scope 3 emissions (supply chain) account for 70-80% of a company's footprint but are invisible to consumers. People want to buy responsibly but can't access supply chain data. Product carbon labels (EU mandatory by 2026) exist but are hard to understand — a score of "2.3 kg CO₂e" means nothing without context.
Proposed Solution
A barcode/QR scanner that queries a database of product carbon footprints (OpenFoodFacts, Klima product DB, or scraped ESG reports). AI summarizes the data into a human-readable breakdown: "This cotton t-shirt: 8.2 kg CO₂e (cotton farming: 60%, manufacturing: 25%, transport: 10%, packaging: 5%). Comparable to driving 41 km."
Core Technology
Barcode scanning via Google ML Kit Barcode Scanning (free). Product DB: OpenFoodFacts API (free, 1.5M+ products) + custom product carbon DB for non-food items. LLM for natural language explanation of carbon data.
Google ML Kit OpenFoodFacts API Claude API React Native Expo
Differentiation
No consumer app currently does this at scale. The EU mandatory product carbon labeling (2026) creates massive market timing. Carbon Lens would be the consumer-facing interface for a regulation that brands are scrambling to comply with.
Feasibility
Medium feasibility. OpenFoodFacts covers food well. Non-food products need a custom DB (can start with 100-200 common items for hackathon demo). AI summarization adds the "wow factor" without complex ML.

IDEA #7
GridPulse — Smart Home Carbon Optimizer
Connect smart home devices to real-time grid carbon intensity data and automatically optimize when you run appliances, charge EVs, or heat/cool your home for minimum carbon impact.
IoT / Smart Home Real-time Data AI Scheduling
Problem
Electricity carbon intensity varies 10x throughout the day — from 50 gCO₂/kWh (high renewables) to 500+ gCO₂/kWh (peak fossil). Running your dishwasher at 2 AM vs 6 PM is a 5-10x difference in carbon. Yet most people run appliances whenever convenient, ignoring grid conditions.
Proposed Solution
A smart home dashboard that shows live grid carbon intensity for the user's region (via CO2 Signal / electricityMap API). Integrates with smart plugs, thermostats, and EV chargers via Home Assistant or manufacturer APIs. AI scheduler suggests or auto-schedules high-consumption appliances for low-carbon hours. Includes a "Carbon Savings" counter showing real CO₂e saved vs. running on average grid.
Core Technology
CO2 Signal API (free tier: 1 req/s, real-time grid data for 70+ countries) or electricityMap API. Home Assistant API for device control. Rule-based + LLM-powered scheduling engine.
CO2 Signal API Home Assistant MQTT React Dashboard FastAPI
Differentiation
No consumer app currently optimizes when you use electricity based on carbon intensity. electricityMap is for energy professionals. Smart home integrations exist for cost optimization (e.g., Octopus Agile) but not carbon optimization. This bridges the gap between real-time grid data and consumer action.
Feasibility
Medium feasibility for hackathon. CO2 Signal API is free and easy to integrate. Dashboard UI (live carbon intensity gauge + savings counter) is immediately visual. Smart plug automation is a production feature — hackathon demo can show the scheduler simulation without real devices.

05 — Final Recommendation

Which Idea to Build for the Hackathon?

Recommended Build
CarbonMind — AI Carbon Coach

The highest-scoring idea across all four evaluation criteria.

Innovation (9/10): First consumer-facing app combining LLMs with grounded emission factor RAG. No existing app does this. The conversational UX is genuinely novel — most carbon apps look like forms, CarbonMind looks like a conversation.


Feasibility (8/10): Can be fully demoed in 24 hours with Claude API + curated JSON emission factor database. No ML model training, no sensor integration, no hardware dependencies. The core "wow moment" (type a meal description, get a CO₂e answer) works immediately.


Impact (7/10): Addresses the #1 problem in carbon tracking (manual entry friction). Every other idea in this portfolio depends on some form of data collection — CarbonMind removes that barrier entirely by accepting natural language. The coaching layer drives long-term behavior change, not just one-time calculation.


Technical Depth (8/10): RAG pipeline with emission factors is architecturally non-trivial and demonstrates real engineering thinking — not just a frontend. Judges evaluating "technical depth" will see a proper AI system, not a calculator with a chatbot wrapper.


Recommended Hackathon Stack: React + Vite frontend · Claude API (with prompt caching for cost control) · Local JSON emission factor DB (~200 common items, sourced from Climatiq) · Simple session memory for conversation context · Deployed on Vercel.


Secondary recommendation: Pair CarbonMind (main submission) with a FoodPrint calculator widget as a side feature. This demonstrates vision AI on top of the same base — showing breadth without doubling complexity.
